# 多角色与角色记忆功能设计方案

## 1. 项目调研总结

### 1.1 SillyTavern 项目架构

SillyTavern 是一个面向高级用户的 LLM 前端应用，采用以下技术栈：
- **后端**: Node.js + Express.js
- **前端**: 原生 JavaScript (ES Modules) + jQuery + Handlebars
- **存储**: 文件系统 (JSON/JSONL) + PNG 元数据存储角色卡

**核心功能模块**:
- **角色系统**: 支持 TavernCard V2/V3 规范，角色数据嵌入 PNG 图片元数据
- **记忆系统**: 作为扩展实现，自动总结对话历史维护长期记忆

### 1.2 ainft-chat-opcnclaw 项目现状

**技术栈**:
- **框架**: Next.js 16 + React 19 + TypeScript
- **数据库**: PostgreSQL + Drizzle ORM
- **状态管理**: Zustand
- **UI**: Ant Design + LobeHub UI

**现有数据模型**:
- `agents`: AI 助手/角色定义
- `sessions`: 会话管理
- `topics`: 话题/子会话
- `messages`: 消息存储
- `userMemories`: 用户记忆系统（已存在多层记忆结构）

## 4. 核心功能流程设计

### 4.1 多角色对话流程

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant UI as 前端 UI
    participant API as API 层
    participant CS as 上下文服务
    participant PS as 提示词服务
    participant LLM as LLM 服务
    participant DB as 数据库

    User->>UI: 发送消息
    UI->>API: POST /api/chat/message
    API->>DB: 保存用户消息
    
    API->>CS: 构建角色上下文
    CS->>DB: 查询角色配置
    CS->>DB: 查询角色记忆
    CS->>PS: 构建提示词
    PS->>PS: 注入系统提示词
    PS->>PS: 注入记忆上下文
    PS->>LLM: 发送请求
    LLM-->>PS: 返回回复
    PS-->>CS: 返回生成内容
    CS-->>API: 返回完整响应
    API->>DB: 保存 AI 消息
    API-->>UI: 返回消息
    UI-->>User: 显示回复
```

### 4.2 角色记忆生成与检索流程

```mermaid
sequenceDiagram
    autonumber
    participant Msg as 消息流
    participant MS as 记忆服务
    participant ES as Embedding 服务
    participant LLM as LLM 服务
    participant DB as 数据库

    rect rgb(230, 245, 255)
        Note over Msg,DB: 记忆生成流程
        Msg->>MS: 触发记忆生成
        MS->>DB: 获取最近消息
        MS->>LLM: 请求生成总结
        LLM-->>MS: 返回总结内容
        MS->>ES: 生成向量
        ES-->>MS: 返回向量
        MS->>DB: 保存到 character_memories
    end

    rect rgb(255, 245, 230)
        Note over Msg,DB: 记忆检索流程
        Msg->>MS: 请求相关记忆
        MS->>ES: 生成查询向量
        ES-->>MS: 返回向量
        MS->>DB: 向量相似度搜索
        DB-->>MS: 返回候选记忆
        MS->>MS: 按重要性/时间过滤
        MS->>MS: 合并相关记忆
        MS-->>Msg: 返回记忆上下文
    end
```

## 11. 总结

本设计方案基于 SillyTavern 的多角色和记忆功能，结合 ainft-chat-opcnclaw 现有的技术栈，提出了完整的实现方案：

1. **数据层**: 使用 Drizzle ORM 定义清晰的数据库 Schema，支持角色、记忆等核心实体
2. **服务层**: 设计独立的 CharacterService、MemoryService 等核心服务
3. **API 层**: 提供完整的 RESTful API 支持
4. **前端层**: 使用 Zustand 进行状态管理，支持响应式 UI 更新

关键特性：
- 每个角色拥有独立的记忆系统，支持短期/长期记忆分离
- 完整的向量搜索支持，实现语义化记忆检索
