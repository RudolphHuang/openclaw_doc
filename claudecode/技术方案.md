# AINFT API 多格式接入技术方案

## 1. 需求概述

### 1.1 背景

当前AINFT API已支持 `/v1/chat/completions` (OpenAI格式) 和 `/v1/messages` (Claude格式)
的模型调用，但在未付费用户访问控制、模型元数据准确性等方面需要完善。

### 1.2 新需求

| 序号 | 需求                                   | 
|----|--------------------------------------|
| 1  | 未付费用户调用Premium模型时给出合适的报错信息           | 
| 2  | `supported_endpoint_types` 字段根据供应商调整 |
| 3  | `models`接口中 `owned_by` 字段改成正确的provider |

### 2.2 请求处理流程

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Router as Router
    participant Auth as 认证中间件
    participant Distribute as 分发中间件
    participant Relay as Relay控制器
    participant Channel as 渠道适配器
    participant Upstream as 上游服务

    Client->>Router: POST /v1/messages (Claude格式)
    
    Note over Router: extAuthChatRouter<br/>外部鉴权路由组
    
    Router->>Auth: ExternalAuth()
    Auth->>Auth: x-api-key解析API Key
    Auth->>Auth: 调用外部鉴权服务<br/>POST /v1/internal/auth/verify
    Auth-->>Router: 设置Context<br/>user_id=0 (外部用户)
    
    Router->>Distribute: Distribute()
    Distribute->>Distribute: 检查token_model_limit
    Distribute->>Distribute: 选择渠道
    Distribute-->>Router: 设置Channel信息
    
    Router->>Relay: Relay(c, RelayFormatClaude)
    Relay->>Relay: 解析Claude请求
    Relay->>Channel: 获取渠道适配器
    
    alt Claude渠道
        Channel->>Channel: 直接转发Claude格式
    else 其他渠道
        Channel->>Channel: Claude格式→OpenAI格式转换
    end
    
    Channel->>Upstream: 转发请求
    Upstream-->>Channel: 返回响应
    Channel-->>Relay: 返回标准格式
    Relay-->>Client: 返回Claude格式响应
```

## 3. 需求详细设计

### 3.1 需求1：未付费用户Premium模型访问控制

#### 3.1.1 业务流程

```mermaid
flowchart TD
    A[用户请求<br/>/v1/chat/completions<br/>或/v1/messages] --> B{外部鉴权}
    B -->|成功| C[设置Context<br/>id=0, external_user_id=xxx]
    B -->|失败| Z[返回401未授权]
    
    C --> D[Distribute中间件]
    D --> E{检查模型限制}
    
    E -->|token_model_limit_enabled=false| F[允许访问所有模型]
    E -->|token_model_limit_enabled=true| G{模型在允许列表?}
    
    G -->|是| H[允许访问]
    G -->|否| I{Premium模型?}
    
    I -->|是| J[返回403错误<br/>Premium模型需要付费订阅]
    I -->|否| K[返回403错误<br/>无权访问该模型]
    
    F --> L[选择渠道]
    H --> L
    L --> M[转发请求]
```

#### 3.1.2 关键逻辑

| 检查点         | 位置                            | 处理方式                   |
|-------------|-------------------------------|------------------------|
| 外部用户识别      | `middleware/external_auth.go` | `id=0` 表示外部鉴权用户        |
| 模型限制检查      | `middleware/distributor.go`   | 检查 `token_model_limit` |
| Premium模型判断 | 需新增配置                         | 根据模型标签或供应商判断           |
| 错误返回        | `middleware/distributor.go`   | 返回OpenAI格式错误响应         |

#### 3.1.3 错误响应格式

```json
{
  "error": {
    "message": "Access restricted. Deposit required to unlock premium models. ",
    "type": "api_error",
    "code": "access_denied"
  }
}
```

### 3.2 需求2：supported_endpoint_types字段调整

#### 3.2.1 当前端点类型定义

```mermaid
classDiagram
    class EndpointType {
        <<enumeration>>
        openai
        anthropic
        gemini
    }

    class ChannelType {
        <<enumeration>>
        ChannelTypeAnthropic
        ChannelTypeGemini
        ChannelTypeSiliconFlow
        ChannelTypeOpenAI
        ...
    }

    class Model {
        +string ModelName
        +EndpointType[] SupportedEndpointTypes
        +string OwnedBy
    }

    ChannelType --> EndpointType: 映射
    Model --> EndpointType: 支持
```

#### 3.2.2 端点类型映射关系

| 渠道类型        | 支持的EndpointTypes  | 说明 |
|-------------|-------------------|------|
| Anthropic   | openai, anthropic | 原生支持Claude格式，平台同时提供OpenAI格式转换 |
| Gemini      | openai, anthropic | 原生支持Gemini格式，平台同时提供OpenAI/Claude格式转换 |
| SiliconFlow | openai, anthropic | **原生仅支持OpenAI格式**，通过平台格式转换层支持Claude格式接入 |
| OpenAI      | openai, anthropic | 原生支持OpenAI格式，平台同时提供Claude格式转换 |

> **关于SiliconFlow的说明**：根据调研，硅基流动API完全兼容OpenAI规范（`https://api.siliconflow.cn/v1`），不直接提供原生Claude格式。平台通过`Format Converter`组件将Claude格式请求转换为OpenAI格式后转发给硅基流动，再将响应转换回Claude格式返回。因此`supported_endpoint_types`包含`anthropic`是合理的，体现了平台的统一接入能力。即使SiliconFlow提供了Gemini模型，事实上我们拿到的接口也是OpenAI格式的。

### 3.3 需求3：owned_by字段修正

#### 3.3.1 当前问题

当前`owned_by`字段来源于`adaptor.GetChannelName()`，返回的是渠道名称（如"claude"、"siliconflow"），而非实际的模型供应商。

#### 3.3.2 供应商映射规则

模型与供应商的映射关系如下：

| 模型ID模式     | 供应商(owned_by) |
|------------|---------------|
| gpt-*      | openaI        |
| claude-*   | anthropic     |
| gemini-*   | google        |
| minimax-*  | siliconflow   |

#### 3.3.3 修正方案

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Controller as Model Controller
    participant Pricing as Pricing Service
    participant Vendor as Vendor Service
    
    Client->>Controller: GET /v1/models
    Controller->>Pricing: GetPricing()
    Pricing->>Pricing: 构建模型列表
    
    loop 每个模型
        Pricing->>Vendor: 根据模型名匹配供应商
        Vendor->>Vendor: 匹配defaultVendorRules
        Vendor-->>Pricing: 返回VendorID和VendorName
        Pricing->>Pricing: 设置OwnerBy=VendorName
    end
    
    Pricing-->>Controller: 返回模型列表
    Controller-->>Client: 返回带正确owned_by的响应
```
