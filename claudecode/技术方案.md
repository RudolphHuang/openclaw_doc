# Claude API 适配技术方案

## 1. 概述

本文档描述在 AINFT 平台上实现 Claude API 协议适配的技术方案。通过该适配层，Claude 协议客户端可以无缝使用 AINFT 平台提供的各类大模型服务。

## 3. 核心流程

### 3.1 请求处理流程

```mermaid
flowchart TD
    Start([收到 Claude 请求]) --> ValidateJSON{校验 JSON 格式}
    ValidateJSON -->|无效| Error400[返回 400 错误]
    ValidateJSON -->|有效| ValidateModel{校验 model 字段}
    
    ValidateModel -->|缺失| Error400Model[返回 400 错误]
    ValidateModel -->|存在| NormalizeModel[模型别名转换]
    
    NormalizeModel --> ResolveProvider{解析 Provider}
    ResolveProvider -->|失败| Error404[返回 404 错误]
    ResolveProvider -->|成功| ValidateMessages{校验 messages}
    
    ValidateMessages -->|缺失| Error400Msg[返回 400 错误]
    ValidateMessages -->|有效| NormalizeStream[校验 stream 参数]
    
    NormalizeStream -->|无效| Error400Stream[返回 400 错误]
    NormalizeStream -->|有效| ConvertRequest[Claude → OpenAI 请求转换]
    
    ConvertRequest --> AuthHeader[处理认证头<br/>x-api-key → Authorization]
    AuthHeader --> ForwardRequest[转发到 Provider 路由]
    
    ForwardRequest --> HandleResponse{处理响应}
    HandleResponse -->|SSE 流式| ConvertStream[流式响应转换]
    HandleResponse -->|JSON 非流式| ConvertJSON[JSON 响应转换]
    HandleResponse -->|错误| ConvertError[错误响应转换]
    
    ConvertStream --> EndStream([返回 Claude SSE 流])
    ConvertJSON --> EndJSON([返回 Claude JSON])
    ConvertError --> EndError([返回 Claude 格式错误])
```

#### 3.2.2 流式请求

```mermaid
sequenceDiagram
    participant Client as Claude 客户端
    participant Route as /v1/messages
    participant Handler as Claude Handler
    participant Transform as SSE 转换流
    participant Provider as Provider 路由
    participant Upstream as 上游模型服务

    Client->>Route: POST /v1/messages?stream=true
    Note right of Client: stream: true
    
    Route->>Handler: handleMessagesPOST(req)
    Handler->>Handler: 协议转换
    Handler->>Provider: 转发请求
    
    Provider->>Upstream: 建立流式连接
    Upstream-->>Provider: SSE 数据流
    
    Provider-->>Handler: SSE 流
    Handler->>Transform: 创建转换管道
    
    Note over Transform: 实时转换 SSE 事件
    
    Transform-->>Client: event: message_start
    Transform-->>Client: event: content_block_start
    
    loop 每个文本块
        Upstream-->>Transform: event: text<br/>data: "片段"
        Transform-->>Client: event: content_block_delta<br/>data: {text: "片段"}
    end
    
    Upstream-->>Transform: event: stop
    Transform-->>Client: event: message_delta<br/>data: {stop_reason: "end_turn"}
    Transform-->>Client: event: message_stop
```

## 11. 注意事项

### 11.1 协议差异处理

| Claude 协议特性 | 处理方式 |
|---------------|---------|
| system 作为独立字段 | 转换为 OpenAI 的 system message |
| content block 数组 | 根据类型分别转换 |
| tool_use / tool_result | 映射为 tool_calls / tool 消息 |
| stop_reason 枚举值 | 建立映射表转换 |
| x-api-key 认证 | 转换为 Authorization: Bearer |

### 11.2 边界情况

1. **空消息处理**: 校验并返回 400 错误
2. **无效 stream 参数**: 支持布尔值、字符串、数字，无效值返回 400
3. **上游 500 错误**: 脱敏处理后返回通用错误信息
4. **SSE 解析失败**: 优雅降级，保持连接
5. **模型别名未找到**: 透传原始模型名，由 Provider 路由处理
